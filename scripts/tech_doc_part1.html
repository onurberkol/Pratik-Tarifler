Proje Genel Bakış

1.1 Vizyon ve Problem Tanımı

Pratik Tarifler, kullanıcının evindeki mevcut malzemelerden yola çıkarak yapay zeka ile tarif öneren bir mobil uygulamadır. Çözdüğü temel problem şudur: her gün milyonlarca insan "akşam ne pişirsem?" sorusuyla karşılaşır ve bu karar yorgunluğu hem zaman kaybına hem de gıda israfına yol açar.

Uygulamanın ayırt edici özelliği 3 modlu keşif sistemidir. Kullanıcı, niyetine göre farklı bir mod seçer:

  • Mod 1 — Evdeki Kalanlar: Buzdolabı fotoğrafı çekilir, AI malzemeleri tanır, sadece tamamı elde olan tarifler önerilir.
  • Mod 2 — Hızlı Tedarik: Eldeki malzemelere ek 1-3 malzemeye kadar izin verilir, eksikler ve tahmini fiyatlar gösterilir.
  • Mod 3 — Sınırsız Keşif: 2500 tariflik havuzda filtre tabanlı arama.
🎯 Teknik Hedefler
Düşük gecikme (öneri <1sn), ölçeklenebilir backend (10K+ MAU), çevrimdışı destek, 13 dil, %99.5+ çökmesiz oturum oranı.

Sistem Mimarisi

2.1 Katmanlı Mimari

Sistem dört ana katmandan oluşur. Veri akışı yukarıdan aşağıya doğrudur; istemci katmanı asla doğrudan veritabanına yazmaz — tüm yazma işlemleri Cloud Functions üzerinden geçer.

① İSTEMCİ KATMANI — React Native + Expo
17 ekran · 10 custom hook · 5 paylaşılan component · i18n (13 dil) · Çevrimdışı SQLite önbellek · Push bildirim
▼ HTTPS / Firebase SDK
② API KATMANI — Cloud Functions (Node.js / TypeScript)
getRecipeRecommendations (3 mod) · analyzePantryPhoto · scanProductLabel · similarRecipes · subscription webhook'ları · oran sınırlama
▼ Admin SDK
③ VERİ KATMANI — Firebase
Firestore (recipes_{lang}, users, ratings) · Cloud Storage (görseller) · Authentication · 13 bileşik indeks
▼ REST / SDK
④ HARİCİ SERVİSLER
Google Vision API (görsel tanıma) · OpenAI (embedding) · RevenueCat (IAP) · Stripe (web ödeme) · Unsplash/Pexels/Pixabay (görsel kaynak)
🔒 Güvenlik Prensibi
İstemci katmanı Firestore'a yalnızca okuma erişimine sahiptir. Tüm yazma işlemleri (favori, puan, abonelik) Cloud Functions üzerinden doğrulanır. Bu, veri bütünlüğünü garanti eder ve kötüye kullanımı engeller.
