Çeviri Pipeline'ı (Faz 3.3)

Uygulama 13 dil destekler. 2500 tarifin 12 hedef dile çevrilmesi — başlık, malzemeler, adımlar dahil — devasa bir iştir. Bu, AI tabanlı bir çeviri pipeline'ı ile otomatikleştirilir.

6.1 Glossary (Terim Sözlüğü) Sistemi

Naif çeviri Türk yemek kültürünü bozar. "Mantı" kelimesi İngilizceye "ravioli" diye çevrilmemelidir — özgün kalmalıdır. Pipeline bunu bir glossary ile çözer:

  • 67 Türk yemek ismi orijinal bırakılır (mantı, baklava, simit, lahmacun...)
  • 10 pişirme fiili her dile doğru karşılığıyla eşlenir (kavurmak, haşlamak...)
  • 8 ölçü birimi yerelleştirilir (su bardağı, çay kaşığı...)
✓ Kültürel Doğruluk
Glossary, çeviri kalitesini garanti eder. "Su böreği" İngilizcede "Su Böreği (Turkish water pastry)" biçiminde — özgün isim korunarak, parantez içinde açıklama eklenerek çevrilir.

6.2 Çeviri Motoru ve Maliyet

Çeviri için Claude Haiku modeli kullanılır — hızlı, ucuz ve kültürel bağlamda güçlüdür. Pipeline, tarifleri batch'ler hâlinde işler ve glossary'yi her prompt'a enjekte eder.

KapsamDillerTahmini Maliyet
MVPEN, DE, AR (3 dil)~$9
Tam12 hedef dil~$35

Hedef diller: İngilizce, Almanca, Fransızca, İtalyanca, İspanyolca, Portekizce, Yunanca, Hollandaca, Rusça, Sırpça, Arapça, İbranice. Arapça ve İbranice için uygulama otomatik RTL (sağdan sola) düzene geçer. 

# Çeviri pipeline çalıştırma
# MVP — sadece 3 dil
python translate_recipes.py --langs en,de,ar --input recipes_db_FULL_tr.json

# Tam — 12 dil
python translate_recipes.py --langs all --batch-size 20

7.2 3 Modlu UX Mimarisi

Üç mod, tek bir birleşik API endpoint'i (getRecipeRecommendations) üzerinden çalışır. İstemci yalnızca mode parametresini değiştirir; backend dispatcher doğru algoritmaya yönlendirir.

MOD 1 — PANTRY (Evdeki Kalanlar)
Girdi: malzeme listesi → Çıktı: yalnızca TÜM malzemesi mevcut tarifler · match_percentage=100
MOD 2 — SUPPLY (Hızlı Tedarik)
Girdi: liste + max_missing(1-3) → Çıktı: eksik malzemeli tarifler + fiyat tahmini · en az eksikten sıralı
MOD 3 — DISCOVER (Sınırsız Keşif)
Girdi: filtreler (mutfak, öğün, diyet, süre) → Çıktı: sayfalanmış tüm havuz · trend/yeni/puan sıralama

7.4 Öneri Algoritması — Derinlemesine

Mod 1 ve 2'nin teknik zorluğu şudur: Firestore, "tüm bu malzemeler şu tarifte var mı?" sorgusunu doğrudan desteklemez. Çözüm iki aşamalıdır:

Aşama 1 — Aday Havuzu (Firestore tarafı)

array-contains-any operatörü ile kullanıcının herhangi bir malzemesini içeren tarifler çekilir. Bu, geniş ama hızlı bir ön filtredir. 

// Firestore sorgusu — aday havuzu
recipes_tr
  .where('image_status', '==', 'ready')
  .where('ingredient_tokens', 'array-contains-any', userTokens)
  .orderBy('rating_avg', 'desc')
  .limit(200)  // Mod 1 için aday havuzu

Aşama 2 — Kesin Filtre (İstemci/Function tarafı)

Aday havuzu bellekte hassas biçimde filtrelenir. Mod 1 için: tarifin tüm token'ları kullanıcıda olmalıdır. Mod 2 için: eksik sayısı max_missing sınırını aşmamalıdır. 

// Mod 1 — kesin eşleşme kontrolü
const userSet = new Set(ingredients);
for (const recipe of candidates) {
  const allCovered = recipe.ingredient_tokens
    .every(token => userSet.has(token));
  if (allCovered) {
    recipe.match_percentage = 100;
    matches.push(recipe);
  }
}

// Mod 2 — eksik sayısı hesabı
const missing = recipe.ingredient_tokens
  .filter(token => !userSet.has(token));
if (missing.length <= maxMissing) {
  recipe.missing_count = missing.length;
  recipe.missing_ingredients = missing.map(toLabel);
  matches.push(recipe);
}
// Sıralama: en az eksikten en çoğa
matches.sort((a,b) => a.missing_count - b.missing_count);
⚙️ array-contains-any Sınırı
Firestore bu operatörde en fazla 30 değer kabul eder. Kullanıcı 30'dan fazla malzeme girerse, en sık kullanılan 30 token sorgulanır; geri kalan istemci tarafında işlenir.
