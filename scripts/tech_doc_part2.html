3.2 Tarif Veri Şeması

Her tarif aşağıdaki yapıda saklanır. ingredient_tokens alanı, öneri algoritmasının kalbidir — normalize edilmiş malzeme kimlikleridir.

// Recipe nesnesi (TypeScript tipi)
{
  id: "tr-mercimek-corbasi",
  title: "Mercimek Çorbası",
  cuisine: "turkish",
  difficulty: "easy",
  total_time_min: 35,
  servings: 4,
  ingredient_tokens: ["lentil", "onion", "carrot", "flour"],
  ingredients: [{ name: "kırmızı mercimek", amount: 1, unit: "cup" }, ...],
  steps: [{ order: 1, text: "Soğanı kavurun..." }, ...],
  diet_tags: ["vegetarian", "vegan"],
  meal_type: ["soup", "lunch"],
  is_premium: false,
  image: { full: "...", thumb: "...", blur_hash: "L6Pj..." },
  rating_avg: 4.8, rating_count: 0, view_count: 0
}
✓ Token Sistemi Neden Önemli?
Kullanıcı "kırmızı mercimek" yazsa da "mercimek" arasa da aynı lentil token'ına eşlenir. Bu normalize katman, Mod 1/2 öneri algoritmasının doğru çalışmasını sağlar — 60+ malzeme token'ı tanımlıdır.

4.1 Firestore Koleksiyon Yapısı

Tarifler her dil için ayrı koleksiyonda tutulur (recipes_tr, recipes_en, ...). Bu, dil bazlı sorgu performansını artırır. Görseller ise dilden bağımsızdır — tüm diller aynı görsel URL'sini paylaşır. 

// Firestore koleksiyon ağacı
recipes_tr/{recipeId}          // 2500 doküman, dil başına
recipes_en/{recipeId}
  ...
users/{userId}
  ├── favorites/{recipeId}
  ├── scan_history/{scanId}
  └── quotas/{date}            // günlük scan limiti takibi
ratings/{userId}_{recipeId}    // kullanıcı puanları
analytics_events/{eventId}     // olay günlüğü
📊 13 Bileşik İndeks
Firestore çoklu-alan sorguları için bileşik indeks gerektirir. Örnek: cuisine + rating_avg, meal_type + total_time_min, ingredient_tokens + image_status. Tümü firestore.indexes.json'da tanımlıdır ve firebase deploy ile yüklenir.

Görsel Üretim Pipeline'ı (Faz 3.2)

2500 tarifin her birine yüksek kaliteli bir görsel gerekir. Bunları tek tek manuel bulmak haftalar sürerdi. Çözüm: otomatik batch (toplu) işleme pipeline'ı. Bu bölüm, görsel içeriğin nasıl üretildiğini adım adım açıklar.

5.1 Hibrit Kaynak Stratejisi

Pipeline, dört görsel kaynağını öncelik sırasıyla dener. Ücretsiz stok fotoğraf kaynakları önce denenir; hiçbiri yeterli kalitede sonuç vermezse, son çare olarak yapay zeka ile görsel üretilir.

💡 Neden Hibrit?
Türk yöresel yemekleri (örn. "perde pilavı", "kuru fasulye") stok fotoğraf sitelerinde nadiren bulunur. Bu tarifler için DALL-E 3, "geleneksel Türk sunumu" yönergesiyle özgün görsel üretir. Yaygın yemekler (pizza, salata) ise ücretsiz kaynaklardan gelir — bu, maliyeti minimumda tutar.

5.2 Tek Tarif İçin İşlem Adımları

Pipeline her tarif için şu 6 adımı uygular:

  1. Arama: Tarifin search_terms alanından sorgu üretilir (örn. "lentil Turkish food") ve kaynaklar sırayla taranır.
  2. Kalite skoru: Bulunan görsel çözünürlük, en-boy oranı ve alaka açısından puanlanır. Skor ≥ 0.70 ise kabul edilir.
  3. AI yedek: Hiçbir kaynak eşik değeri geçemezse DALL-E 3 devreye girer.
  4. Dönüştürme: Görsel 1200×900 WebP (kalite 85) + 400×300 thumbnail'e çevrilir.
  5. BlurHash: ~20 karakterlik bulanık placeholder string üretilir (progressif yükleme için).
  6. Kayıt: Sonuç results.json'a yazılır; görseller Storage'a yüklenmeye hazır hâle gelir.
# Pipeline çalıştırma komutları
# 1. Dry-run — API çağrısı yapmadan simülasyon (test)
python image_pipeline_v2.py --dry-run

# 2. Pilot batch — 50 tarif, 13 mutfak temsili
python image_pipeline_v2.py --input pilot_batch.json --output ./output/

# 3. Tek tarif testi
python image_pipeline_v2.py --recipe-id tr-mercimek-corbasi

# 4. Sadece AI — önceki başarısız işleri yeniden dene
python image_pipeline_v2.py --ai-only --input failed_jobs.json

5.3 Batch İş Kuyruğu Yönetimi

2500 görsel tek seferde işlenmez. image_jobs.json dosyası, her tarif için bir iş kaydı tutar. Bu kayıt durumu (pending, processing, completed, failed) ve deneme geçmişini izler. Pipeline kesilse bile kaldığı yerden devam eder. 

// image_jobs.json — tek iş kaydı yapısı
{
  job_id: "img_job_tr-mercimek-corbasi",
  recipe_id: "tr-mercimek-corbasi",
  status: "pending",         // pending|processing|completed|failed
  attempts: [],                // her denemenin günlüğü
  priority: 2,                // 1=premium öncelikli
  requires_review: false,     // AI üretimi insan onayı bekler
  search_terms: {
    primary_keywords: "lentil",
    cuisine_keyword: "Turkish food",
    fallback_query: "lentil Turkish food"
  }
}
